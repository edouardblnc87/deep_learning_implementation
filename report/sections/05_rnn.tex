\section{Recurrent Neural Network}
\label{sec:rnn}

% ──────────────────────────────────────────────────────────────────────────────
\subsection{From FNN to RNN}

The FNN section identified two structural limitations: \emph{order blindness}
(the lag vector has no positional encoding, so temporal ordering is only
partially exploited through learned weight magnitudes) and \emph{regime
blindness} (the model resets entirely between predictions and cannot accumulate
context across days). A Recurrent Neural Network (RNN) addresses both
by construction.

Three properties of a flat-window FNN make it unsuitable as sequence length
grows:
\begin{itemize}
  \item \textbf{Order blindness.} Each lag receives an independently learned
        fixed weight; shuffling the input features degrades performance but
        does not break it — temporal ordering is an optimisation artefact,
        not a structural guarantee.
  \item \textbf{Fixed hard cutoff.} The window size must be chosen upfront;
        any dependency beyond it is invisible to the model.
  \item \textbf{Parameter explosion.} Every additional lag adds a full row of
        input weights, making long contexts expensive.
\end{itemize}

An RNN processes the sequence one time step at a time, maintaining a
\emph{hidden state} $\mathbf{h}_t \in \mathbb{R}^{H}$ (where $H$ is the hidden
size) that acts as a compressed memory of all previous inputs:
\[
  \mathbf{h}_t
  = \tanh\!\bigl(W_{ih}\,\mathbf{x}_t + W_{hh}\,\mathbf{h}_{t-1} + \mathbf{b}\bigr),
  \qquad \mathbf{h}_0 = \mathbf{0},
\]
where $W_{ih} \in \mathbb{R}^{H \times d}$ maps the current input,
$W_{hh} \in \mathbb{R}^{H \times H}$ is the \emph{recurrence matrix} that
propagates the hidden state, and $d$ is the input dimension.
Temporal ordering is now structural: the same $W_{ih}$ and $W_{hh}$ are
applied at every step, so the position of $\mathbf{x}_t$ in the sequence
determines which historical context $\mathbf{h}_{t-1}$ it is combined with.

\paragraph{Vanishing gradient problem.}
During backpropagation through time (BPTT), the gradient with respect to a
hidden state $T-t$ steps in the past is multiplied by $(W_{hh}^\top)^{T-t}$.
If the spectral radius $\rho(W_{hh}) < 1$, this product shrinks
exponentially and long-range gradients vanish; if $\rho(W_{hh}) > 1$, they
explode. In practice, $\tanh$ saturation further attenuates gradients, limiting
effective credit assignment to roughly 10--20 steps.
Gradient clipping is the standard palliative: it caps the gradient norm before
each update, preventing explosion without curing vanishing.

\subsection{Architecture and Training Setup}

\subsubsection*{Architecture}

The model feeds a sequence of $T = 20$ scalar inputs (one-day log-returns)
into a single-layer RNN with hidden size $H = 64$ and $\tanh$ nonlinearity,
then maps the final hidden state $\mathbf{h}_T$ to a scalar volatility
forecast through a linear head:
\[
  \bigl(x_1, \ldots, x_{20}\bigr)
  \;\xrightarrow{\;\text{RNN}_{64,\tanh}\;}\;
  \mathbf{h}_{20} \in \mathbb{R}^{64}
  \;\xrightarrow{\;\text{Linear}\;}\;
  \hat{y} \in \mathbb{R}_{+}.
\]
The head uses Xavier uniform initialisation and a zero bias; no dropout is
applied to the baseline.

\subsection{Vanishing Gradient Diagnosis}

% ──────────────────────────────────────
\subsubsection{$W_{hh}$ Spectral Radius}

During BPTT, the gradient signal flowing back $k$ steps is multiplied by
$(W_{hh}^\top)^k$. The fate of each gradient direction is determined by the
corresponding singular value $\sigma_i$ of $W_{hh}$: if $\sigma_i < 1$ the
contribution of that direction vanishes; if $\sigma_i > 1$ it explodes.

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/figure_516.png}
  \caption{Singular value spectrum of $W_{hh}$ after training.
    Spectral radius $\rho = 1.4$ (exploding regime overall), but roughly
    57 of the 64 directions lie below~1. Training has deliberately pushed
    5--7 directions above~1 to maintain gradient flow through the tanh
    saturation; gradient clipping prevents these from causing instability.}
  \label{fig:rnn_spectrum}
\end{figure}

The spectral radius of the trained $W_{hh}$ is $\rho = 1.4$, placing the
model in the exploding-gradient regime in aggregate. However, approximately
57 of the 64 singular values lie below~1; only 5--7 directions were
deliberately pushed above~1 by training. This is the only way the model can
maintain gradient flow over 20 steps of $\tanh$ saturation: a handful of
amplifying directions compensate for the many vanishing ones.
Gradient clipping (threshold 0.5) is therefore not merely a safety measure
--- it is a necessary component of stable training given this spectral
structure.

% ──────────────────────────────────────
\subsubsection{Output Sensitivity per Time Step}

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/figure_517.png}
  \caption{Output sensitivity to the hidden state at each time step:
    $\partial\hat{y}/\partial\mathbf{h}_t$ as a function of lag.
    Sensitivity decays by 50\,\% at step~1 and is essentially zero by
    step~2 --- the prediction is nearly insensitive to any hidden state
    older than one step.}
  \label{fig:rnn_grad_norms}
\end{figure}

The output sensitivity to $\mathbf{h}_t$ collapses within one step:
50\,\%-decay occurs at lag~1 and the signal is negligible by lag~2.
This is the backward view of vanishing gradients --- the Jacobian of the
output with respect to past hidden states decays rapidly.
This seems to contradict the input sensitivity result (Section~5.4.3), which
showed that perturbations to $\mathbf{x}_t$ at all lags affect the output.
The next section resolves this apparent contradiction.

% ──────────────────────────────────────
\subsubsection{Effective Memory}

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/figure_518.png}
  \caption{Effective memory: correlation between the final hidden state
    $\mathbf{h}_T$ and each input $\mathbf{x}_t$ at different lags.
    Despite the near-zero gradient norms beyond lag~2, the correlation
    decays gradually across the full 20-lag window and remains meaningful
    at lags 15--20.}
  \label{fig:rnn_effective_memory}
\end{figure}

The correlation between the final hidden state $\mathbf{h}_T$ and each
input $\mathbf{x}_t$ decays gradually across the full 20-step window and
remains non-negligible at lags 15--20. This resolves the apparent
contradiction with Section~5.6.2: the two metrics measure different things.

\begin{itemize}
  \item \textbf{Output sensitivity} (Section~5.6.2) measures the local linear
        derivative $\partial\hat{y}/\partial\mathbf{h}_t$: how much the output
        changes if the hidden state at step~$t$ is infinitesimally perturbed.
        This collapses because the Jacobian of the recurrence is small in most
        directions.
  \item \textbf{Effective memory} (here) measures the global nonlinear
        correlation between $\mathbf{h}_T$ and $\mathbf{x}_t$: whether the
        information from past inputs is encoded in the final state, regardless
        of the local derivative.
\end{itemize}

A function can be strongly correlated with a variable while having a near-zero
local derivative at the current operating point.
The forward dynamics of the RNN do accumulate historical context into
$\mathbf{h}_T$; it is the backward gradient path that fails to
propagate credit for that information.

\end{itemize}

\subsection{Hyperparameter Grid Search}

A grid search over 48 configurations was run to validate the baseline
architecture choices:
\begin{align*}
  &\text{seq\_len} \in \{10, 20, 30\}, \quad
   H \in \{32, 64\}, \quad
   \text{layers} \in \{1, 2\}, \\
  &\alpha\,(\text{learning rate}) \in \{5{\times}10^{-4},\, 10^{-3}\}, \quad
   \text{clip} \in \{0.5, 1.0\}.
\end{align*}
Each configuration was trained with a light budget (80 epochs, early stopping
at patience~15) to allow broad coverage.

Two conclusions emerge from the grid search:

\paragraph{Architecture is the ceiling, not hyperparameters.}
The spread from rank~1 to rank~48 is only $0.00042$ in validation MAE ---
essentially nothing.
Tuning hidden size, number of layers, learning rate, or gradient clipping
produces negligible gains. The model's performance ceiling is set by its
architecture, not by any hyperparameter combination within this search space.

\paragraph{Sequence length matters, but only for the spikes regime.}
Sequence length~10 is consistently weak on the spikes regime
(MAE~$= 0.015$--$0.017$); sequence length~30 clusters around
MAE~$\approx 0.008$ on spikes --- a meaningful improvement.
Crucially, this benefit arises from \emph{forward-pass accumulation}, not
from improved gradient-based learning: as shown in Section~5.6.2, gradients
do not flow back beyond lag~2, but the forward dynamics do encode a
longer history into $\mathbf{h}_T$ when more input steps are provided.
The longer sequence helps because the COVID crash was a sustained event;
seeing more of it in context improves the hidden state at inference time.
The trade-off is a small degradation on the calm regime, where longer
windows introduce additional noise from distant, uninformative lags.

    \bottomrule
  \end{tabular}

The RNN achieves \textbf{MAE~$= 0.502$} (standardised) versus \textbf{0.579}
for the best FNN --- a \textbf{13\,\%} improvement --- by exploiting temporal
ordering and accumulating context across the 20-step sequence.
Three diagnostic findings characterise the model's behaviour and limitations:

\begin{enumerate}
  \item \textbf{Mean-reversion bias is an output-range problem, not a
        context problem.} The model fails on extreme days not because it
        lacks historical information (the crash context is clearly encoded
        in the hidden state) but because it never encountered targets of
        that magnitude during training and therefore never learned to
        output them.
  \item \textbf{Forward dynamics accumulate history; backward gradients do
        not.} The effective memory (correlation-based) spans 20 lags; the
        gradient-based sensitivity collapses within 2 steps. Longer sequences
        improve inference by enriching the forward hidden state, even though
        the recurrence matrix cannot propagate credit for that information
        backward.
  \item \textbf{Architecture, not hyperparameters, is the binding
        constraint.} A 48-configuration grid search produces a spread of
        only $0.00042$ MAE. The vanilla RNN is structurally limited: the
        final prediction depends almost entirely on $\mathbf{h}_T$, which
        is the hidden state most strongly influenced by the most recent step.
        The model sees 30 lags of history but learns primarily from the last
        one.
\end{enumerate}

These structural limitations suggest that a gated architecture such as the LSTM
is a natural next step, as its cell state is specifically designed to address
the gradient collapse identified here.
